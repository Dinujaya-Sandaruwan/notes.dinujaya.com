C# NOTES

01. Introduction to C# programming language

C# is a modern, object-oriented programming language developed by Microsoft. It is commonly used to build desktop applications, web applications, and games for Windows devices.

Here are some basic concepts you should know about C#:

  • C# is strongly typed, which means you must declare the data type of a variable before you use it.
  • C# supports object-oriented programming, which means you can define classes, interfaces, and other objects to encapsulate data and behavior.
  • C# uses a syntax similar to other C-style languages, such as C++ and Java.

Here is an example of declaring and using a variable in C#:

// Declare a string variable
string greeting;
// Assign a value to the variable
greeting = "Hello, world!";
// Print the value to the console
Console.WriteLine(greeting);

Output:

Hello, world!

02. Understanding the .NET framework

The .NET framework is a software framework developed by Microsoft that provides a way to build and run applications on the Windows operating system. It includes a large class library, a runtime environment, and various tools for building, testing, and deploying applications.

Here are some basic concepts you should know about the .NET framework:

  • The .NET framework provides a common language runtime (CLR), which manages memory, security, and other aspects of running code on the Windows operating system.
  • The .NET framework includes a large class library, which provides a wide range of pre-built classes and APIs for common programming tasks.
  • The .NET framework supports multiple programming languages, including C#, Visual Basic, and F#.

Here is an example of using the .NET framework to create a simple console application in C#:

// Declare a namespace for the application
      namespace HelloWorld
{
    // Declare a class for the application
    class Program
    {
        // Define the entry point for the application
        static void Main(string[] args)
        {
            // Print a greeting to the console
            Console.WriteLine("Hello, world!");
        }
    }
}

Output:

Hello, world!

05. Data types and variables in C#

C# supports a wide range of data types, including numeric, character, boolean, and reference types. Here are some of the most commonly used data types:

  • int: a 32-bit signed integer
  • double: a double-precision floating-point number
  • char: a Unicode character
  • bool: a Boolean value, which can be either true or false
  • string: a sequence of Unicode characters

To declare a variable in C#, you need to specify the data type and a name for the variable. Here are some examples:

// Declare an integer variable
int age;
// Assign a value to the integer variable
age = 30;

// Declare and assign a value to a double variable
double height = 1.75;

// Declare and assign a value to a string variable
string name = "Alice";

You can also declare and assign a value to a variable in a single statement:

// Declare and assign a value to a boolean variable
bool isStudent = true;

Once you have declared a variable, you can use it in your program. For example:

// Declare and assign values to two integer variables
int x = 5;
int y = 3;
// Calculate the sum of the two integer variables
int sum = x + y;

// Print the sum to the console
Console.WriteLine("The sum of {0} and {1} is {2}", x, y, sum);

Output:

The sum of 5 and 3 is 8

In this example, we declare two integer variables x and y, assign them the values 5 and 3, respectively, and then calculate their sum and assign it to the variable sum. Finally, we print the sum to the console using the Console.WriteLine() method and string interpolation.

Understanding data types and variables is crucial for writing C# programs. Experiment with different data types and variable declarations to gain a better understanding of how they work.

06. Operators and expressions in C#

Operators are symbols that perform operations on one or more operands. In C#, operators are used to perform arithmetic, comparison, logical, and bitwise operations. Here are some examples:

  • Arithmetic operators: + (addition), - (subtraction), * (multiplication), / (division), % (modulus)
  • Comparison operators: == (equality), != (inequality), < (less than), <= (less than or equal to), > (greater than), >= (greater than or equal to)
  • Logical operators: && (logical AND), || (logical OR), ! (logical NOT)
  • Bitwise operators: & (bitwise AND), | (bitwise OR), ^ (bitwise XOR), ~ (bitwise NOT), << (left shift), >> (right shift)

Expressions are combinations of operators, operands, and method calls that produce a value. Here are some examples:

// Arithmetic expression
int sum = 5 + 3;
// Comparison expression
bool isGreater = 5 > 3;

// Logical expression
bool isTrue = (5 > 3) && (2 < 4);

// Bitwise expression
int result = 5 & 3;

You can also use parentheses to group subexpressions and control the order of evaluation:

// Expression with parentheses
int result = (5 + 3) * 2;

Understanding operators and expressions is essential for writing C# programs. Experiment with different operators and expressions to gain a better understanding of how they work.

07. Control flow statements in C#

Control flow statements are used to control the flow of execution in a C# program. There are three main types of control flow statements: conditional statements, loop statements, and jump statements.

Conditional statements

Conditional statements are used to execute different blocks of code based on the value of a Boolean expression. The most common conditional statements in C# are if, else if, and else. Here's an example:

int x = 5;
if (x > 0)
{
  Console.WriteLine("x is positive");
}
else if (x < 0)
{
  Console.WriteLine("x is negative");
}
else
{
  Console.WriteLine("x is zero");
}

Loop statements

Loop statements are used to execute a block of code repeatedly until a certain condition is met. The most common loop statements in C# are for, while, and do while. Here's an example:

// For loop
for (int i = 0; i < 10; i++)
{
  Console.WriteLine(i);
}
// While loop
int j = 0;
while (j < 10)
{
  Console.WriteLine(j);
  j++;
}

// Do while loop
int k = 0;
do
{
  Console.WriteLine(k);
  k++;
} while (k < 10);

Jump statements

Jump statements are used to transfer control to another part of the program. The most common jump statements in C# are break, continue, and return. Here's an example:

// Break statement
for (int i = 0; i < 10; i++)
{
  if (i == 5)
  {
    break;
  }
  Console.WriteLine(i);
}
// Continue statement
for (int j = 0; j < 10; j++)
{
if (j == 5)
{
continue;
}
  Console.WriteLine(j);
}

// Return statement
int Sum(int a, int b)
{
return a + b;
}

Understanding control flow statements is essential for writing C# programs. Experiment with different conditional statements, loop statements, and jump statements to gain a better understanding of how they work.

08. Working with arrays and collections

Arrays and collections are used to store multiple values of the same type in a single variable. In C#, arrays and collections are both used for this purpose, but they have different characteristics and usage patterns.

Arrays

An array is a fixed-size data structure that stores a collection of elements of the same type. To create an array in C#, you need to specify the type of the elements and the size of the array. Here's an example:

// Create an integer array of size 5
int[] myArray = new int[5];
// Initialize the array
myArray[0] = 1;
myArray[1] = 2;
myArray[2] = 3;
myArray[3] = 4;
myArray[4] = 5;
      
// Accessing array elements
Console.WriteLine(myArray[2]); // Output: 3

You can also use an array initializer to initialize the values of an array at the time of its creation. Here's an example:

// Create an integer array using an array initializer
int[] myArray = new int[] { 1, 2, 3, 4, 5 };
// Accessing array elements
Console.WriteLine(myArray[2]); // Output: 3

Collections

A collection is a dynamic-size data structure that stores a collection of elements of the same or different types. In C#, collections are implemented using classes from the System.Collections and System.Collections.Generic namespaces. Here's an example of using the List class:

// Create a list of integers
List<int> myList = new List<int>();
// Add elements to the list
myList.Add(1);
myList.Add(2);
myList.Add(3);
myList.Add(4);
myList.Add(5);
      
// Accessing list elements
Console.WriteLine(myList[2]); // Output: 3

The List class provides many useful methods for working with collections, such as Add, Remove, Insert, and Contains.

Arrays and collections are powerful tools for managing large sets of data in your C# programs. Experiment with different types of arrays and collections to learn more about their characteristics and usage patterns.

09. Methods and functions in C#

Methods and functions are used to encapsulate a block of code that performs a specific task. In C#, methods and functions are used interchangeably and have the same syntax. A method or function can take zero or more parameters, and it can return a value of a specific type or void (no value).

Creating a method or function

To create a method or function in C#, you need to specify the access modifier (public, private, protected, or internal), the return type (if any), the name of the method or function, and the parameters (if any). Here's an example of a simple method that takes two integers and returns their sum:

// Define a method that takes two integers and returns their sum
public int AddNumbers(int num1, int num2)
{
    int sum = num1 + num2;
    return sum;
}
// Call the method and print the result
int result = AddNumbers(3, 5);
Console.WriteLine(result); // Output: 8

You can also create a function that does not return a value by using the void keyword as the return type:

// Define a function that takes a string parameter and prints it to the console
public void PrintMessage(string message)
{
    Console.WriteLine(message);
}
// Call the function
PrintMessage("Hello, world!"); // Output: Hello, world!

Passing arguments by value or by reference

When you pass an argument to a method or function, you can choose to pass it by value or by reference. By default, C# passes arguments by value, which means that a copy of the argument is passed to the method or function. If you pass an argument by reference, the method or function can modify the original value of the argument. Here's an example:

// Define a method that takes an integer parameter by reference and modifies its value
public void IncrementNumber(ref int num)
{
    num++;
}
// Call the method and print the result
int myNumber = 5;
IncrementNumber(ref myNumber);
Console.WriteLine(myNumber); // Output: 6

In this example, the IncrementNumber method takes an integer parameter by reference using the ref keyword. When the method is called, it modifies the value of the original argument, which is reflected in the output.

Methods and functions are essential building blocks of any C# program. By encapsulating code in methods and functions, you can make your programs more modular and easier to understand and maintain.

10. Debugging your code in Visual Studio

Debugging is the process of finding and fixing errors in your code. Visual Studio provides a powerful set of debugging tools that can help you identify and solve problems in your C# programs. Here's a quick overview of how to use the debugging tools in Visual Studio:

Setting breakpoints

A breakpoint is a marker in your code that tells Visual Studio to pause execution of the program at that point. You can set a breakpoint by clicking in the left margin of the code editor, or by pressing F9 on your keyboard. When the program reaches the breakpoint, execution will pause and you can inspect the state of the program.

int[] numbers = { 1, 2, 3, 4, 5 };
int sum = 0;
// Set a breakpoint on this line
for (int i = 0; i < numbers.Length; i++)
{
sum += numbers[i];
}
      
Console.WriteLine(sum);

Stepping through code

Once the program is paused at a breakpoint, you can use the debugging tools to step through the code line by line. The F10 key steps over the current line of code, while the F11 key steps into a method or function call.

int[] numbers = { 1, 2, 3, 4, 5 };
int sum = 0;
for (int i = 0; i < numbers.Length; i++)
{
  // Step into this line of code
  sum += numbers[i];
}
      
Console.WriteLine(sum);

Inspecting variables

While debugging, you can inspect the value of variables in your program by hovering over them in the code editor or by using the Locals or Watch windows. The Locals window shows all variables in the current scope, while the Watch window allows you to manually specify variables to track.

int[] numbers = { 1, 2, 3, 4, 5 };
int sum = 0;
for (int i = 0; i < numbers.Length; i++)
{
  sum += numbers[i];
}

// Add a watch on the sum variable

By using breakpoints, stepping through code, and inspecting variables, you can quickly identify and fix errors in your C# programs. Debugging is an essential skill for any developer, and Visual Studio provides a powerful set of tools to help you get the job done.

11. Object-oriented programming in C#

Object-oriented programming (OOP) is a programming paradigm that organizes code into objects that interact with each other. C# is an object-oriented programming language that supports many OOP concepts, including classes, objects, inheritance, and polymorphism.

Classes and objects

A class is a blueprint for creating objects. It defines the properties and methods that an object of that class will have. To create an object of a class, you use the new keyword followed by the name of the class:

// Define a class
class Person
{
    public string Name;
    public int Age;
}
// Create an object of the Person class
Person person = new Person();
person.Name = "Alice";
person.Age = 30;

Inheritance

Inheritance is a way to create a new class that is a variation of an existing class. The new class, called the subclass or derived class, inherits all the properties and methods of the existing class, called the superclass or base class, and can add or modify its own properties and methods. To create a subclass, you use the : symbol followed by the name of the base class:

// Define a base class
class Animal
{
    public string Name;
}
// Define a subclass that inherits from Animal
class Dog : Animal
{
public string Bark()
{
return "Woof!";
}
}

// Create an object of the Dog class
Dog dog = new Dog();
dog.Name = "Fido";
string bark = dog.Bark();

Polymorphism

Polymorphism is the ability of objects of different classes to be treated as if they were objects of the same class. This is achieved through inheritance and interfaces. An interface defines a contract that a class must implement, and allows objects of different classes to be treated as if they implement the same interface:

// Define an interface
interface IAnimal
{
    string Name { get; }
}
// Implement the interface in the Dog class
class Dog : IAnimal
{
public string Name { get; set; }
public string Bark()
{
    return "Woof!";
}
}

// Create objects of the Dog and Cat classes and treat them as IAnimal
IAnimal animal1 = new Dog();
IAnimal animal2 = new Cat();

Object-oriented programming is a powerful paradigm for organizing and structuring code. C# provides a rich set of features for implementing OOP concepts, including classes, objects, inheritance, and polymorphism. By mastering these concepts, you can write more organized, maintainable, and extensible code.

13. Inheritance and polymorphism in C#

Inheritance is a mechanism in object-oriented programming that allows you to define a new class based on an existing class. The new class inherits the properties and methods of the existing class, and can add new properties and methods or override existing ones.

In C#, you define a new class that inherits from an existing class using the colon (:) operator, like this:

// Define a base class
class Animal
{
    public virtual void MakeSound()
    {
        Console.WriteLine("The animal makes a sound.");
    }
}
// Define a derived class that inherits from Animal
class Dog : Animal
{
  public override void MakeSound()
  {
    Console.WriteLine("The dog barks.");
  }
}

In this example, the Animal class has a method called MakeSound that prints a generic message to the console. The Dog class inherits from Animal and overrides the MakeSound method to print a message specific to dogs.

Polymorphism is another important concept in object-oriented programming that allows you to treat objects of different classes as if they were of the same class. In C#, you can achieve polymorphism using inheritance and interfaces.

Here's an example of how to use polymorphism in C#:

// Define a list of animals
List<Animal> animals = new List<Animal>();
// Add a dog and a cat to the list
animals.Add(new Dog());
animals.Add(new Cat());

// Make each animal in the list make a sound
foreach (Animal animal in animals)
{
  animal.MakeSound();
}

This code creates a list of animals and adds a dog and a cat to it. It then uses a foreach loop to call the MakeSound method of each animal in the list. Since both the dog and the cat classes inherit from the Animal class and override its MakeSound method, the correct sound is played for each animal.

Inheritance and polymorphism are powerful tools in C# that allow you to write modular and reusable code. By using inheritance and polymorphism, you can create hierarchies of classes that model real-world relationships, and write code that is more flexible and easier to maintain.

14. Interfaces and abstract classes

Interfaces and abstract classes are important concepts in object-oriented programming that allow you to define common behavior and contracts for classes that share a common interface or inheritance hierarchy.

An interface is a contract that defines a set of properties, methods, and events that a class must implement. In C#, you define an interface using the interface keyword, like this:

interface IAnimal
{
    void MakeSound();
}

This interface defines a single method called MakeSound that any class that implements the interface must implement. You can use interfaces to define a common contract that multiple classes can implement, without forcing them to inherit from a common base class.

An abstract class is a class that cannot be instantiated, but can be inherited from. Abstract classes can define abstract methods that must be implemented by any derived class. In C#, you define an abstract class using the abstract keyword, like this:

abstract class Animal
{
    public abstract void MakeSound();
}

This abstract class defines an abstract method called MakeSound that any derived class must implement. You can use abstract classes to define a common interface that multiple classes can inherit from, while providing some basic implementation details.

Here's an example of how to use interfaces and abstract classes in C#:

// Define an interface for animals that can fly
interface IFlyingAnimal
  {
    void Fly();
  }
  // Define an abstract class for animals
  abstract class Animal
  {
    public string Name { get; set; }
    public int Age { get; set; }
    public abstract void MakeSound();
  }

  // Define a derived class for birds that can fly
  class Bird : Animal, IFlyingAnimal
  {
    public void Fly()
  {
    Console.WriteLine("The bird is flying.");
  }
  public override void MakeSound()
  {
    Console.WriteLine("The bird is chirping.");
  }
}

In this example, we define an interface called IFlyingAnimal that defines a method called Fly. We also define an abstract class called Animal that defines properties for the name and age of an animal, and an abstract method called MakeSound.

We then define a derived class called Bird that inherits from Animal and implements IFlyingAnimal. The Bird class provides an implementation of the Fly method and overrides the MakeSound method.

By using interfaces and abstract classes, you can define common behavior and contracts for classes that share a common interface or inheritance hierarchy. This makes it easier to write modular and reusable code that can be extended and customized as needed.

19. Delegates and Events in C#

First, let's define a delegate that represents a method with a specific signature:

public delegate void MyDelegate(string message);

Next, let's create a class that uses this delegate:

public class MyClass
{
    public event MyDelegate MyEvent;

    public void RaiseEvent(string message)
    {
        if (MyEvent != null)
        {
            MyEvent(message);
        }
    }
}

In this example, `MyClass` has an event named `MyEvent` that uses the `MyDelegate` delegate. The `RaiseEvent` method raises the event by invoking the delegate if it's not null.

Now, let's create a client class that subscribes to the event:

public class MyClient
{
    public void HandleEvent(string message)
    {
        Console.WriteLine(message);
    }
}
      
MyClass myClass = new MyClass();
MyClient myClient = new MyClient();
      
myClass.MyEvent += new MyDelegate(myClient.HandleEvent);
      
myClass.RaiseEvent("Hello, world!"); // Output: Hello, world!

In this example, `MyClient` defines a method named `HandleEvent` that handles the event by writing the message to the console. `MyClass` creates an instance of `MyClient`, subscribes to the `MyEvent` event using the `+=` operator, and raises the event by calling the `RaiseEvent` method.

Delegates and events are powerful tools for creating flexible and extensible code in C#. By defining delegates and events, you can allow other classes to register for notifications when something interesting happens, and you can decouple the implementation of the event from its subscribers.

21. Advanced LINQ techniques

Language Integrated Query (LINQ) is a powerful tool that allows you to perform complex queries on various data sources, such as collections, databases, and XML files. In addition to basic LINQ operations, such as filtering, sorting, and grouping, there are several advanced techniques that can be used to make your code more efficient and expressive.

1. Deferred Execution

One of the most powerful features of LINQ is its ability to perform deferred execution. This means that LINQ queries are not executed immediately, but are instead executed only when the results are needed. This allows you to write code that is more efficient, because LINQ will only retrieve the data that is actually required. 

var query = from c in customers
where c.City == "London"
select c;
foreach (var c in query)
{
    Console.WriteLine(c.Name);
}

In this example, the LINQ query is not executed until the foreach loop is run. If there are a large number of customers in the list, this can save a significant amount of memory and processing time.

2. Concatenating and Combining Queries

Another useful feature of LINQ is its ability to combine multiple queries into a single query. This can be done using the Concat and Union methods. 

var query1 = from c in customers
where c.City == "London"
  select c;
var query2 = from c in customers
where c.City == "Paris"
  select c;
var combinedQuery = query1.Concat(query2);
foreach (var c in combinedQuery)
{
    Console.WriteLine(c.Name);
}

In this example, the Concat method is used to combine the results of two queries into a single query.

3. Aggregating Data

LINQ provides several methods for aggregating data, such as Sum, Count, and Average. These methods allow you to quickly and easily calculate the total, count, and average of a set of values. 

var prices = new List<decimal> { 10.5m, 20.4m, 15.8m, 30.6m };
var totalPrice = prices.Sum();
var averagePrice = prices.Average();
var count = prices.Count();
Console.WriteLine("Total Price: " + totalPrice);
Console.WriteLine("Average Price: " + averagePrice);
Console.WriteLine("Count: " + count);

In this example, the Sum, Average, and Count methods are used to calculate the total price, average price, and count of a list of prices.

26. Introduction to web services and REST APIs

Web services and REST APIs are a way for applications to communicate with each other over the internet. They provide a standard way of exchanging data and making use of remote functionality.

A web service is a software system designed to support interoperable machine-to-machine interaction over a network. It has an interface described in a machine-processable format (specifically WSDL), usually an XML schema, that other systems can interact with. The interaction is described using SOAP protocol, which is based on XML.

A REST API is a type of web service that uses HTTP protocol for exchanging data. It relies on a stateless, client-server architecture where data and functionality are separated. RESTful APIs use standard HTTP methods like GET, POST, PUT, and DELETE to perform operations on resources. The resources are identified by URLs, and the data is often exchanged in JSON or XML format.

Here's an example of a REST API call using the HTTP GET method:

using System;
using System.Net.Http;

class Program
{
    static async System.Threading.Tasks.Task Main(string[] args)
    {
        var client = new HttpClient();

        var response = await client.GetAsync("https://jsonplaceholder.typicode.com/posts/1");

        var content = await response.Content.ReadAsStringAsync();

        Console.WriteLine(content);
    }
}

This code makes an HTTP GET request to a RESTful API endpoint at "https://jsonplaceholder.typicode.com/posts/1", which returns a JSON object representing a blog post. The code reads the response content as a string and outputs it to the console.

With web services and REST APIs, applications can leverage the power of the internet to share data and functionality. They enable developers to build applications that are more flexible, scalable, and interoperable.

29. Code profiling and optimization techniques in C#

Profiling is the process of analyzing the performance of your code to identify areas that can be optimized. In C#, there are several profiling and optimization techniques that you can use to improve the performance of your code.

Profiling tools

C# provides several built-in profiling tools, including the Visual Studio Profiler and the dotTrace Profiler from JetBrains. These tools allow you to analyze the performance of your code at runtime and identify areas that are causing performance bottlenecks.

Code optimization techniques

Once you've identified areas that need optimization, there are several techniques you can use to improve the performance of your C# code:

Use the right data structures and algorithms

Choosing the right data structures and algorithms can have a significant impact on the performance of your code. For example, using a HashSet instead of a List for membership tests can result in a significant performance improvement.

Avoid unnecessary object creation

C# is a garbage-collected language, which means that creating and destroying objects can be an expensive operation. Try to avoid unnecessary object creation by reusing existing objects or using value types instead of reference types where possible.

Minimize I/O operations

I/O operations, such as reading from and writing to disk or network resources, are typically slower than other operations in C#. Minimize I/O operations by buffering data or using asynchronous I/O operations.

Profile and optimize performance-critical code

Identify code that is critical to the performance of your application and profile it to identify areas that can be optimized. For example, you might use a profiler to identify bottlenecks in your code and then optimize the code to eliminate the bottlenecks.

Code profiling and optimization techniques are essential for improving the performance of your C# applications. By using profiling tools and optimization techniques, you can identify areas that need improvement and optimize your code for better performance.
