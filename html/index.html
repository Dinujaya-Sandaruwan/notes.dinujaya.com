CSS NOTES

01. What is CSS?

CSS stands for Cascading Style Sheets. It is a style sheet language used for describing the presentation of a document written in HTML or XML. CSS separates the content from presentation, making it easy to maintain and style documents on the web. Here's an example of how CSS is applied:


          /* Selects all h1 elements */
          h1 {
            font-size: 2em;
            color: #333;
            text-align: center;
          }

In the example above, the CSS selects all the <h1> elements and applies the styles defined within the curly braces. The font-size, color, and text-align properties are set to 2em, #333, and center respectively. This will make all <h1> elements on the page have a font size of 2em, color of #333, and centered text alignment.

02. CSS History, Present & Future

CSS was first introduced in 1996 by the World Wide Web Consortium (W3C). The first version of CSS, known as CSS1, was very basic and had limited styling capabilities. However, it paved the way for the development of modern CSS.

Over the years, CSS has undergone several updates and enhancements, with CSS2 being released in 1998 and CSS3 in 1999. CSS3 is the latest version of CSS and includes new features like animations, gradients, and flexbox layouts.

The future of CSS looks promising with ongoing development and updates. The latest version, CSS4, is currently in development and is expected to bring even more advanced features to CSS, including native support for variables, nesting, and more.

Here's an example of using CSS3 to create a simple gradient background:


          /* Applies a linear gradient background to the body element */
          body {
            background: linear-gradient(to bottom, #fff, #ccc);
          }

The above CSS code will create a gradient background for the <body> element, with the gradient going from white (#fff) at the top to light gray (#ccc) at the bottom.

03. Adding CSS to our Project with Inline Styles

There are several ways to add CSS to an HTML document. One of the easiest ways is by using inline styles. Inline styles are defined within the HTML tag using the style attribute.

Here's an example of adding inline styles to a <p> element:


          <p style="font-size: 1.2em; color: #333;">This is a paragraph with inline styles.</p>

In the above example, the style attribute is added to the <p> tag with two CSS properties: font-size and color. The value for font-size is set to 1.2em and the value for color is set to #333.

Inline styles are useful when you want to apply styles to individual elements, but they can become difficult to manage if you have a lot of styles or multiple pages with similar styles. In those cases, it's better to use an external stylesheet or internal styles within the <head> tag.

04. Understanding the <style> Tag & Creating a .css File

The <style> tag is used to define styles within an HTML document. It can be used in two ways:

  1. Inline styles: The <style> tag can be added within the <head> tag to define styles for the entire document.
  2. External stylesheets: The <style> tag can be used to link to an external stylesheet file that contains all the CSS styles for the document.

Here's an example of using the <style> tag to define styles for a <p> element:


          <head>
            <style>
              p {
                font-size: 1.2em;
                color: #333;
              }
            </style>
          </head>
          <body>
            <p>This is a paragraph with styles defined in the <style> tag.</p>
          </body>

In the above example, the <style> tag is added within the <head> tag and contains CSS styles for the <p> element. The styles defined within the <style> tag will be applied to all <p> elements within the document.

Alternatively, you can create an external CSS file with all the styles and link to it using the <link> tag. Here's an example:


          <head>
            <link rel="stylesheet" href="styles.css">
          </head>

In the above example, the <link> tag is used to link to an external CSS file named "styles.css". This file should contain all the CSS styles for the document.

05. Applying Additional Styles & Importing Google Fonts

Now that we know how to add CSS styles to our HTML document, let's look at some additional styles we can apply and how to import external fonts using Google Fonts.

Here's an example of applying some additional styles to a <div> element:


          div {
            background-color: #f5f5f5;
            border: 1px solid #ddd;
            padding: 10px;
          }

In the above example, we've added a background color, a border, and some padding to a <div> element. These styles will be applied to all <div> elements within the document.

Next, let's look at how to import external fonts using Google Fonts. Here's an example:


          <head>
            <link rel="preconnect" href="https://fonts.googleapis.com">
            <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
            <link href="https://fonts.googleapis.com/css2?family=Open+Sans:wght@400;700&display=swap" rel="stylesheet">
            <style>
              body {
                font-family: 'Open Sans', sans-serif;
              }
            </style>
          </head>

In the above example, we've added three <link> tags to import the Google Fonts stylesheet. The first two <link> tags are used for preconnecting to Google Fonts and improving the font loading performance. The third <link> tag is used to import the Open Sans font family with two weights: 400 and 700.

Finally, we've added a <style> tag within the <head> tag to apply the Open Sans font family to the entire document's body.

06. CSS Selectors

CSS selectors are used to target specific HTML elements and apply styles to them. Here are some examples of CSS selectors:

  • Element Selector: This targets all instances of a specific HTML element. For example:
    • p {
        color: red;
      }
  • ID Selector: This targets a specific HTML element with a unique ID. For example:
    • #header {
        background-color: blue;
      }
  • Class Selector: This targets all instances of a specific class in the HTML document. For example:
    • .container {
        max-width: 800px;
      }
  • Attribute Selector: This targets HTML elements based on their attributes. For example:
    • a[href="https://www.google.com"] {
        color: green;
      }
  • Pseudo-class Selector: This targets HTML elements based on their state, such as :hover or :active. For example:
    • a:hover {
        text-decoration: underline;
      }

By combining these selectors with CSS properties and values, we can apply styles to specific HTML elements and create visually appealing websites.

07. Understanding the "Cascading" Style & Specificity

The term "cascading" in Cascading Style Sheets (CSS) refers to the way in which styles are applied to HTML elements. In simple terms, when multiple styles are applied to the same element, the browser determines which styles to use based on their specificity and the order in which they appear in the CSS code.

Specificity refers to how "specific" a style selector is. The more specific a selector is, the higher its priority will be. Here are some examples of selectors and their specificity:

  • Element Selector: Has a specificity of 1.
  • Class Selector: Has a specificity of 10.
  • ID Selector: Has a specificity of 100.
  • Inline Style: Has a specificity of 1000.

When two or more selectors have the same specificity, the one that appears last in the CSS code will take precedence. Here's an example:


          p {
            color: blue;
          }
          p {
            color: red;
          }

In the above example, all <p> elements will have a red color, as the second rule appears last in the CSS code.

Understanding specificity and the cascading style order is important for writing efficient and maintainable CSS code.

08. Understanding Inheritance

CSS inheritance is the process by which styles are passed down from parent elements to their child elements. This means that if a parent element has a certain style applied to it, that style will automatically be applied to its child elements unless they have their own styles defined.

For example, consider the following HTML code:


    <div class="parent">
      <p>This is some text</p>
    </div>

If we define a style for the .parent class like this:


    .parent {
      color: blue;
    }

Then the <p> element inside the <div class="parent"> will also have a blue color, unless it has its own style defined. This is because the <p> element inherits the color property from its parent element.

It's important to note that not all CSS properties are inherited. For example, properties related to layout or positioning are typically not inherited. To see if a particular property is inherited, you can check the CSS documentation for that property.

Understanding inheritance can help you write more efficient and maintainable CSS code by reducing the amount of redundant code you need to write.

09. Adding Combinators

CSS combinators are used to select elements based on their relationship to other elements. There are four types of combinators in CSS:

  • Descendant Selector (space): Selects an element that is a descendant of another element.
  • Child Selector (>): Selects an element that is a direct child of another element.
  • Adjacent Sibling Selector (+): Selects an element that is directly after another element.
  • General Sibling Selector (~): Selects all elements that come after another element.

Here's an example of using the descendant selector to select all <p> elements inside a <div> element:


          div p {
            color: blue;
          }

This will select all <p> elements that are descendants of a <div> element, regardless of how deeply they are nested.

Here's an example of using the child selector to select only the direct <p> children of a <div> element:


          div > p {
            color: blue;
          }

This will select only the <p> elements that are direct children of a <div> element.

Using combinators can help you write more targeted and specific CSS selectors, which can make your styles more efficient and easier to maintain.

10. Introducing the CSS Box Model

The CSS box model is a fundamental concept in web design that defines how HTML elements are displayed on a webpage. Every HTML element is essentially a rectangular box that consists of four parts:

  • Content: The actual content of the element, such as text or images.
  • Padding: The space between the content and the element's border.
  • Border: The line that surrounds the element's padding and content.
  • Margin: The space between the element's border and neighboring elements.

Here's an example of how the box model is represented in CSS:


    div {
      width: 200px;
      height: 100px;
      padding: 20px;
      border: 2px solid black;
      margin: 10px;
    }

This code will create a <div> element that is 200 pixels wide and 100 pixels tall. It will have 20 pixels of padding, a 2 pixel border that is solid and black, and 10 pixels of margin around the entire element.

Understanding the box model is crucial for creating layouts and positioning elements on a webpage. It can also help you troubleshoot layout issues and understand how different CSS properties affect an element's size and position.

11. Understanding Margin Collapsing and Removing Default Margins

Margin collapsing is a phenomenon that occurs when two adjacent elements have vertical margins. In some cases, the margins will "collapse" into a single margin that is equal to the larger of the two margins. This can be confusing and unexpected, but it is an important concept to understand.

To avoid margin collapsing, you can use one of several techniques:

  • Use padding instead of margin on one of the elements.
  • Add a border or padding to one of the elements.
  • Add a non-zero height to one of the elements.
  • Add a non-zero float or position property to one of the elements.

Another important aspect of margins is that many HTML elements have default margins that can interfere with your layout. For example, the <body> element typically has a margin around the entire page, which can cause issues if you're trying to create a full-width layout.

To remove default margins, you can use the following CSS code:


          body {
            margin: 0;
          }

This code will set the margin of the <body> element to zero, effectively removing any default margin. You can also use this technique with other elements that have default margins, such as <p> or <h1>.

12. Working with Shorthand Properties

In CSS, shorthand properties allow you to specify multiple related properties in a single line of code. This can make your code shorter and easier to read, but it's important to understand how shorthand properties work so that you can use them effectively.

Here's an example of a shorthand property:


          margin: 10px 20px 10px 20px;

This code sets the margin of an element to 10 pixels on the top, 20 pixels on the right, 10 pixels on the bottom, and 20 pixels on the left. The order of the values is top, right, bottom, left, going clockwise around the element.

Another example of a shorthand property is the font property:


          font: italic small-caps bold 16px/1.5 Arial, sans-serif;

This code sets several properties related to the font of an element, including the font-style, font-variant, font-weight, font-size, line-height, and font-family. The values are listed in a specific order, and not all values are required.

Using shorthand properties can save you time and help keep your code organized, but it's important to be aware of the properties you're setting and the order in which they're listed.

13. Diving Into the Height & Width Properties

The height and width properties in CSS allow you to set the height and width of an element. These properties are commonly used for layout purposes, such as setting the size of a container or an image.

Here's an example of how to use these properties:


          .container {
            width: 500px;
            height: 300px;
          }

This code sets the width of the element with the class of "container" to 500 pixels and the height to 300 pixels. You can also use other units of measurement, such as percentages or ems, to set the height and width.

It's important to note that the height and width properties only set the dimensions of the content box of an element, not including any padding, border, or margin. If you want to set the size of the entire element, including padding and border, you can use the box-sizing property:


          .container {
            box-sizing: border-box;
            width: 500px;
            height: 300px;
            padding: 20px;
            border: 1px solid black;
          }

This code sets the box-sizing property to border-box, which includes the padding and border in the width and height of the element. This means that the content box will be 500 pixels wide and 300 pixels high, including the 20-pixel padding and 1-pixel border on each side.

14. Understanding Box Sizing

The box-sizing property in CSS allows you to control how the size of an element is calculated, taking into account its padding and border.

The default value of the box-sizing property is content-box, which means that the width and height properties only apply to the content box of the element, not including its padding and border.

If you want to include the padding and border in the width and height of the element, you can set the box-sizing property to border-box:


          .container {
            box-sizing: border-box;
            width: 500px;
            height: 300px;
            padding: 20px;
            border: 1px solid black;
          }

This code sets the box-sizing property to border-box, which means that the width and height properties now include the padding and border of the element. The resulting element will have a content box of 460 pixels wide (500 - 2 * 20 - 2 * 1) and 260 pixels high (300 - 2 * 20 - 2 * 1), with a total width of 500 pixels and a total height of 300 pixels.

It's important to note that changing the box-sizing property can affect the layout of your entire page, so it's important to use it consistently throughout your code.

15. Understanding the Display Property

The display property in CSS controls how an element is displayed on a web page. There are several values that the display property can take:

  • block: the element generates a block-level box.
  • inline: the element generates an inline-level box.
  • inline-block: the element generates a block-level box that will flow with the surrounding text as if it were an inline element.
  • none: the element is completely removed from the page and takes up no space.

The default value for most HTML elements is block, which means that they will take up the entire width of their container and create a new line after them. However, there are some exceptions, such as inline elements like <a> and <span>.

The display property can also be used to create responsive designs by changing the layout of elements depending on the size of the screen. For example, you can set an element to block on desktop screens and inline-block on mobile screens to create a more compact layout.

It's important to note that changing the display property can affect the layout of your entire page, so it's important to use it carefully and test your code across different devices.

16. Styling Anchor Tags

Anchor tags (<a>) are commonly used in web design to create hyperlinks to other pages or resources. To style anchor tags, you can use the :link, :visited, and :hover pseudo-classes.

The :link pseudo-class is used to style links that have not been visited by the user, while the :visited pseudo-class is used to style links that have already been visited. The :hover pseudo-class is used to style links when the user hovers over them with their mouse.

Here's an example of how to style anchor tags:

a:link {
  color: blue;
  text-decoration: none;
}
a:visited {
  color: purple;
}

a:hover {
  text-decoration: underline;
}

In this example, the :link pseudo-class sets the color of unvisited links to blue and removes the underline, while the :visited pseudo-class sets the color of visited links to purple. The :hover pseudo-class underlines the link when the user hovers over it.

You can also style anchor tags using other CSS properties, such as font-size, background-color, and padding, to create custom link styles that match your website's design.

17. Pseudo Classes & Pseudo Elements

Pseudo-classes and pseudo-elements are special selectors in CSS that allow you to style elements based on their state or position in the HTML document.

Pseudo-classes

Pseudo-classes are used to select elements based on their state, such as when a user hovers over an element or clicks on it. Here are some common pseudo-classes:

  • :hover - Selects an element when the user hovers over it with their mouse
  • :active - Selects an element when it is being clicked or activated
  • :focus - Selects an element when it has focus, such as when a user clicks on a form field
  • :nth-child() - Selects elements based on their position in the parent container

Here's an example of how to use the :hover pseudo-class to change the background color of a button when the user hovers over it:

button:hover {
        background-color: #FFD700;
      }

Pseudo-elements

Pseudo-elements are used to select and style parts of an element, such as the first letter of a paragraph or the content after an element. Here are some common pseudo-elements:

  • ::before - Inserts content before the selected element
  • ::after - Inserts content after the selected element
  • ::first-letter - Selects the first letter of the text in the selected element
  • ::first-line - Selects the first line of text in the selected element

Here's an example of how to use the ::before pseudo-element to insert an icon before a button:

button::before {
        content: url(icon.png);
        margin-right: 5px;
      }

In this example, the ::before pseudo-element inserts an image of an icon before the button text and adds a 5-pixel margin to the right of the icon.

18. Working with "font-weight" & "border"

The font-weight property specifies the weight or boldness of the text in an element. The values can range from 100 to 900, with 400 being the default normal weight and 700 being the default bold weight.

/* Set the font weight of all paragraphs to bold */
p {
  font-weight: bold;
}
/* Set the font weight of all headings to normal */
h1, h2, h3 {
  font-weight: normal;
}

The border property allows you to set the border of an element. It has three sub-properties: border-width, border-style, and border-color. You can also use the shorthand property border to set all three sub-properties at once.

/* Set a 2-pixel solid black border around all images */
img {
  border: 2px solid #000;
}
/* Set a dotted 1-pixel red border around all links */
a {
  border: 1px dotted #f00;
}

You can also set different border widths, styles, and colors for each side of an element using the border-top, border-right, border-bottom, and border-left properties.

/* Set a 1-pixel solid blue border on the top and bottom of a div */
div {
  border-top: 1px solid blue;
  border-bottom: 1px solid blue;
}
/* Set a 2-pixel dotted green border on the left and right of an element */
.element {
  border-left: 2px dotted green;
  border-right: 2px dotted green;
}

19. Adding a Background Image

You can add a background image to an element using the background-image property in CSS. The value of the property should be the URL of the image you want to use as the background.

/* Set the background image of the body to a local file */
body {
  background-image: url('image.jpg');
}
/* Set the background image of a div to a remote image */
div {
  background-image: url('https://example.com/image.jpg');
}

You can also use the background-repeat property to control whether the background image is repeated or not, and in which direction it is repeated. The values for this property include repeat, repeat-x, repeat-y, and no-repeat.

/* Set the background image of a div to repeat horizontally */
      div {
        background-image: url('image.jpg');
        background-repeat: repeat-x;
      }

You can use the background-size property to control the size of the background image. The value of the property can be set to auto, cover, or contain, or to a specific length or percentage.

/* Set the background image of a div to cover the entire element */
      div {
        background-image: url('image.jpg');
        background-size: cover;
      }

20. Using Multiple CSS Classes & Combined Selectors

You can apply multiple CSS classes to an HTML element by including all the classes in the class attribute, separated by a space. Each class can have its own styles, which are combined when applied to the element.

<div class="box red">This is a red box</div>
.box {
  width: 100px;
  height: 100px;
  border: 1px solid black;
}
.red {
  background-color: red;
}

In this example, the .box class defines the width, height, and border of the element, while the .red class adds a red background to the element. When both classes are applied to the <div> element, the styles from both classes are combined to create a red box with a border.

You can also use combined selectors to apply styles to elements that meet multiple criteria. For example, the following selector will apply styles to all <a> elements with a class of external that are located within a <nav> element:

nav a.external {
  color: red;
}

Combined selectors can be very powerful and allow you to create very specific styles for specific elements in your HTML structure.

21. Classes or IDs?

When selecting elements in CSS, you have two options: classes and IDs. Both have their uses and are appropriate in different situations.

Classes

Classes are used to group elements that share a common style. You can apply the same class to multiple elements, and each element will inherit the styles defined for that class.

<p class="important">This is an important paragraph.</p>
<p class="important">This is another important paragraph.</p>
.important {
  font-weight: bold;
  color: red;
}

In this example, the .important class is applied to two <p> elements, both of which are styled with a bold font weight and red text color.

IDs

IDs are used to uniquely identify an element on a page. Unlike classes, each ID can only be used once per page, and should be used sparingly to avoid conflicts.

<div id="header">This is the page header.</div>
#header {
  background-color: black;
  color: white;
}

In this example, the #header ID is applied to a <div> element, which is styled with a black background and white text color.

When to Use Classes vs. IDs

As a general rule, you should use classes when you want to apply a style to multiple elements on a page, and IDs when you want to uniquely identify a single element. However, there are exceptions to this rule, and it's ultimately up to you to decide which to use based on the specific needs of your project.

22. (Not) using !important

The !important keyword is used to give a CSS declaration more weight than normal. When an !important declaration is used, it will override any other declarations for that property on that element, regardless of specificity.

While !important can be a useful tool in certain situations, it should be used sparingly and with caution. Overuse of !important can make it difficult to debug and maintain your CSS, and can lead to unexpected and unintended results.

When to Use !important

There are some situations where !important can be a useful tool:

  • When you need to override a style applied by a third-party library or plugin.
  • When you need to ensure that a style is always applied, regardless of specificity.

When Not to Use !important

On the other hand, there are many situations where !important should be avoided:

  • When there are other, more specific selectors that could be used instead.
  • When you are working on a large project where it could be difficult to track down unexpected behavior caused by overuse of !important.

Remember, !important should be used only when necessary and with caution. It's always better to rely on specificity and careful use of selectors to achieve the desired style.

23. Selecting the Opposite with :not()

The :not() pseudo-class allows you to select all elements that do not match a particular selector. This can be especially useful in situations where you want to apply a style to all elements except for a few specific ones.

Syntax

selector:not(other-selector) {
        /* styles */
      }

The :not() pseudo-class takes a single argument, which is a selector that specifies the elements to exclude from the selection.

Examples

Select all paragraphs except the first one:

p:not(:first-of-type) {
        /* styles */
      }

Select all elements with the class "button" except for links:

.button:not(a) {
        /* styles */
      }

Select all list items except for the first and last ones:

li:not(:first-of-type):not(:last-of-type) {
        /* styles */
      }

Remember that the :not() pseudo-class is supported in modern browsers, but older browsers may not support it. As always, it's a good idea to test your code in multiple browsers to ensure cross-browser compatibility.

24. CSS & Browser Support

One of the biggest challenges when working with CSS is ensuring that your styles are properly displayed across different browsers. While modern browsers generally support the same CSS features, older browsers may have limited or no support for some properties or selectors. Here are a few tips to help ensure cross-browser compatibility:

  1. Check the compatibility of CSS features you plan to use on Can I Use to see which browsers support them.
  2. Use feature detection to apply styles selectively based on browser support. For example, you can use the @supports rule to apply styles only if a certain CSS feature is supported by the browser:
@supports (display: grid) {
        /* styles for browsers that support CSS Grid */
      }
  1. Avoid using proprietary prefixes (such as -webkit-, -moz-, -ms-, or -o-) whenever possible, as they are specific to certain browsers and may become obsolete over time.
  2. Use CSS resets or normalizers to ensure consistent styles across different browsers. These tools help to eliminate the default styles that different browsers apply to HTML elements, giving you a clean slate to work with.

By following these tips and staying up-to-date on the latest CSS features and browser support, you can create styles that look great across a wide range of devices and browsers.

25. Understanding the Z-Index

The z-index property is used to specify the stack order of positioned elements that overlap within a document. Elements with a higher z-index value will appear on top of elements with a lower z-index value. Here are some important things to keep in mind when using the z-index property:

  • The z-index property only works on positioned elements (elements with position: absolute, position: relative, position: fixed, or position: sticky).
  • The z-index property can take any integer value (positive or negative).
  • The default value of the z-index property is auto, which means that the element will be positioned in the normal document flow and will not be affected by the z-index values of other elements.
  • The z-index property can be used in conjunction with other CSS properties like opacity and transform, as long as the element is positioned.
  • When two elements overlap and have the same z-index value, the one that appears later in the HTML document (i.e. appears lower in the code) will be placed on top.

Here is an example of how to use the z-index property:

.box {
  position: relative;
  z-index: 1;
}
.box2 {
  position: relative;
  z-index: 2;
}

In this example, the .box2 element will appear on top of the .box element because it has a higher z-index value.

26. CSS Sizes and Units

When styling with CSS, it's important to choose the appropriate size and unit for your properties. Here are some of the most common units used in CSS:

  • px (pixels) - A pixel is a single dot on a screen. This unit is commonly used for specifying font sizes and element dimensions. For example, "font-size: 16px" or "width: 300px".
  • % (percentages) - A percentage is a relative unit that is based on the size of the parent element. For example, "width: 50%" will make an element take up half the width of its parent element.
  • em - An em is a unit that is based on the size of the font that is being used. For example, if the font-size of an element is 16px, then 1em is equal to 16px. This unit is commonly used for setting line-heights and spacing between elements.
  • rem - A rem is similar to an em, but it is based on the size of the root element (i.e. the HTML element) rather than the current element. This unit is commonly used for creating scalable layouts that can be easily adjusted for different screen sizes.
  • vh and vw - These units are based on the size of the viewport (i.e. the visible area of the browser window). 1vh is equal to 1% of the viewport height, while 1vw is equal to 1% of the viewport width. These units are commonly used for creating full-screen layouts or for setting element sizes relative to the viewport.

27. Where Units Matter

When it comes to sizing elements in CSS, choosing the right unit is important to achieve the desired result. Here are some cases where units matter:

  • Font size: The most commonly used unit for font sizes is px. However, using em or rem is recommended because they are relative units and allow for more responsive design. For example:
    • body {
  font-size: 16px;
}
h1 {
  font-size: 2em; /* equivalent to 32px */
}

p {
  font-size: 1rem; /* equivalent to 16px */
}
  • Padding and margin: Padding and margin can be specified in px, em, rem, %, and other units. em and rem are again recommended for responsive design. For example:
    • .box {
    padding: 1em;
    margin: 0.5rem;
}
  • Width and height: The most commonly used units for specifying width and height are px and %. em and rem can also be used, but they are relative units and can cause unexpected results in certain cases. For example:
    • .container {
    width: 100%;
    height: 400px;
}
    
    .box {
    width: 50%; /* of the parent container /
    height: 2em;
}
  • Viewport units: Viewport units are relative to the size of the viewport and can be useful for creating responsive designs. The most commonly used viewport units are vw (viewport width) and vh (viewport height). For example:
    • .box {
    width: 50vw; / 50% of the viewport width /
    height: 50vh; / 50% of the viewport height */
}

By choosing the right unit for each case, you can make your CSS more flexible and responsive to different screen sizes and devices.

28. CSS for Styling Forms

CSS can be used to style HTML forms, including input fields, checkboxes, radio buttons, and buttons.

For example, to style an input field:

input {
    background-color: #f1f1f1;
    border: none;
    padding: 12px 15px;
    margin: 8px 0;
    box-sizing: border-box;
}

The CSS properties used above are:

  • background-color: sets the background color of the input field.
  • border: sets the border of the input field.
  • padding: sets the padding inside the input field.
  • margin: sets the margin outside the input field.
  • box-sizing: sets how the width and height of the input field are calculated (in this case, including the border and padding).

Similarly, checkboxes and radio buttons can be styled:

input[type="checkbox"], input[type="radio"] {
        margin: 4px 0;
      }
      
      input[type="checkbox"] {
        /* style the checkbox */
      }
      
      input[type="radio"] {
        /* style the radio button */
      }

To style a button:

button {
    background-color: #4CAF50;
    color: white;
    padding: 14px 20px;
    margin: 8px 0;
    border: none;
    cursor: pointer;
    width: 100%;
}

The CSS properties used above are:

  • background-color: sets the background color of the button.
  • color: sets the color of the text inside the button.
  • padding: sets the padding inside the button.
  • margin: sets the margin outside the button.
  • border: sets the border of the button.
  • cursor: sets the mouse cursor style when hovering over the button.
  • width: sets the width of the button.

29. Styling Fonts

There are several properties in CSS that can be used to style fonts:

  • font-family: specifies the font(s) to be used for the text
  • font-size: specifies the size of the font
  • font-weight: specifies the weight (boldness) of the font
  • font-style: specifies the style (italic, oblique, etc.) of the font
  • text-transform: specifies the capitalization of the text
  • text-decoration: specifies the decoration of the text (underline, overline, line-through, etc.)
  • text-align: specifies the horizontal alignment of the text
  • line-height: specifies the distance between lines of text

Here's an example of how to use some of these properties:


    p {
        font-family: Arial, sans-serif;
        font-size: 16px;
        font-weight: 400;
        font-style: italic;
        text-transform: uppercase;
        text-decoration: underline;
        text-align: center;
        line-height: 1.5;
    }

This will style all <p> elements on the page with Arial (or a sans-serif font if Arial isn't available), a size of 16 pixels, a normal weight, italic style, uppercase letters, underlined text, centered alignment, and a line height of 1.5 times the font size.

30. Why our Project Should Become Responsive

With the increasing usage of mobile devices, it's more important than ever to ensure that our web projects are responsive. Responsive design means that our website can adapt to different screen sizes and device types, providing a better user experience for all users.

There are many benefits to creating a responsive website:

  1. Improved user experience: Users can access your site from any device, and it will look and function properly.
  2. Increased traffic: Google rewards responsive websites with higher search engine rankings.
  3. Cost-effective: Rather than creating separate websites for desktop and mobile, responsive design allows you to create one website that works for all devices.

In order to make our project responsive, we need to use CSS media queries. Media queries allow us to apply different styles based on the screen size of the device being used. We can use media queries to adjust our layout, font sizes, and even hide or show elements depending on the device screen size.


        /* Example of a media query */
        @media screen and (max-width: 600px) {
          /* Styles applied to devices with a maximum width of 600px */
          body {
            font-size: 14px;
          }
        }

With media queries, we can create a website that looks great on any device, improving user experience and helping to increase traffic to our site.

31. Understanding Hardware Pixels vs. Software Pixels

When we talk about screens and displays, there are two types of pixels: hardware pixels and software pixels.

Hardware pixels are the physical dots that make up a screen, and their size and density are fixed, meaning that a screen with a resolution of 1920 x 1080 pixels will always have 1920 x 1080 physical pixels. On the other hand, software pixels, or logical pixels, are the units used by software to render content on the screen.

One important concept related to software pixels is the device pixel ratio (DPR). This is the ratio between the physical size of a pixel and its logical size. For example, if a screen has a DPR of 2, this means that each logical pixel is represented by 2 physical pixels.

Understanding hardware and software pixels is important when it comes to creating responsive designs that look good on different devices. When we specify dimensions and positions in CSS, we should use logical units like em, rem, or %, rather than physical units like px, because logical units will adjust to the DPR of the device and look consistent across different screens.

32. Comparing the Viewport Metatag (HTML) and Media Queries (CSS)

Both the viewport metatag and media queries are used to make a website responsive.

The viewport metatag is used in HTML to control the width and scaling of the viewport on mobile devices. It specifies the viewport's width, the initial scale, and maximum scale. For example:

<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0">

The above meta tag sets the viewport width to be the same as the device width, and disables zooming.

Media queries, on the other hand, are used in CSS to apply different styles depending on the device width. They allow us to create a responsive design by defining different styles for different devices. For example:

@media only screen and (max-width: 768px) {
  /* Styles for devices with a maximum width of 768px */
}
@media only screen and (min-width: 769px) and (max-width: 1024px) {
  /* Styles for devices with a minimum width of 769px and maximum width of 1024px */
}
@media only screen and (min-width: 1025px) {
  /* Styles for devices with a minimum width of 1025px */
}

The above media queries define three different sets of styles for devices with different screen sizes. The first set of styles applies to devices with a maximum width of 768px, the second set of styles applies to devices with a minimum width of 769px and a maximum width of 1024px, and the third set of styles applies to devices with a minimum width of 1025px.

By using both the viewport metatag and media queries, we can create a fully responsive website that looks great on all devices.

33. Understanding the "viewport" Metatag

The viewport meta tag is used to specify the viewport for mobile browsers. It helps the browser to adjust the page layout according to the size of the device screen.

The viewport meta tag is placed inside the head section of your HTML document. Here is an example:

<head>
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
</head>

The content attribute is used to define the settings for the viewport. In the example above, we set the width of the viewport to the device width and set the initial zoom level to 1.0. This means that the webpage will be displayed with a 1:1 ratio between CSS pixels and device-independent pixels.

By using the viewport meta tag, we can make sure that our web page looks good on all devices, from small smartphones to large desktop screens.

34. Designing Websites "Mobile First"

Mobile-first design is a web design strategy in which mobile devices are the first priority when designing a website. It involves designing the website for smaller screen sizes first and then gradually scaling up to larger screens. This approach has become increasingly popular because mobile devices account for a significant portion of web traffic, and it's essential to ensure that your website is optimized for smaller screens.

One of the main advantages of mobile-first design is that it encourages simplicity and efficiency. It forces designers to focus on the most critical elements of the website and avoid unnecessary clutter that can slow down the page's load time. Additionally, it can also help ensure that the website is more accessible to people with disabilities and older devices.

Mobile-first design can be achieved using responsive design techniques. A responsive website is one that adjusts its layout and content based on the screen size and orientation of the device being used. This is achieved using CSS media queries, which allow you to specify different styles for different screen sizes.

By designing with mobile devices in mind first, you can create a website that is optimized for a wide range of devices and screen sizes, which can ultimately lead to a better user experience and increased engagement.
